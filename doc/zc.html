1. Synopsis

zc [options] unitname

2. Description

zc is the zerolang compiler. It compiles zerolang source files (.z) into C source code (.c), which can then be compiled with a standard C compiler (gcc or clang) to produce a native executable.

The compiler operates on units — each source file defines a unit, and the compiler follows dependencies between units automatically. Compilation starts from the named unit and discovers all referenced units through the Virtual File System (VFS).

The VFS presents a unified source tree to the compiler by mounting the user source directory over the system library directory. User units are found first, shadowing any system units with the same name.

3. Arguments

3.1. unitname (required)

The name of the unit to compile. This must be a valid zerolang unit name: lowercase ASCII letters and digits only. The compiler will look for a file named unitname.z in the source directory.

zc hello --src examples

3.2. -o FILE, --output FILE

Specify the output C filename. By default the output file is named unitname.c in the current directory.

zc hello --src examples -o build/hello.c

3.3. --src DIR

Path to the user source directory. Defaults to the current working directory. The compiler searches this directory for .z source files.

zc myprogram --src /path/to/project

3.4. --system DIR

Path to the system library directory. By default this is auto-detected relative to the compiler installation. The system library provides built-in types (i64, f64, String, etc.), operators, and standard types (Option, array, List, Map).

This flag is rarely needed — use it only when developing the compiler itself or testing against an alternate standard library.

3.5. -v, --verbose

Enable verbose output. When active, the compiler prints compilation progress, settings, and timing information to stderr. This is useful for debugging compilation issues or understanding which files are being processed.

zc hello --src examples -v

Example verbose output:

source directory: examples
system directory: /path/to/lib/system
output file: hello.c
parsing hello...
  compiling: core.z
  resolving dependency: system
  compiling: system.z
  compiling: hello.z
type checking...
type check passed
emitting C...
written hello.c
completed in 0.042s

3.6. --full-typecheck

Type-check all definitions in all units, not just those reachable from main. By default the compiler is demand-driven: it only type-checks definitions that are actually used. This flag forces a complete check, which is useful for library development or finding unused dead code.

3.7. --dump-sql FILE

Write a SQL dump of the compiler's internal state to the specified file. The SQL is SQLite-compatible and includes tables for files, tokens, AST nodes, types, and emitted C lines. Use - to write to stdout.

This is a diagnostic tool for compiler development and debugging.

zc hello --src examples --dump-sql hello.sql
sqlite3 hello.sql "SELECT name, cname FROM types WHERE cname IS NOT NULL"

3.8. --no-color

Disable ANSI color codes in error messages. By default, color is enabled when stderr is a terminal (TTY) and disabled when piped or redirected. Use this flag to force plain text output in all cases.

4. Exit Codes

  • 0 — Compilation succeeded. The output C file has been written.
  • 1 — Compilation failed. One or more errors were found during parsing or type checking. Error messages are written to stderr.
  • 2 — Usage error. Invalid command-line arguments, invalid unit name, or invalid directory paths.

5. Error Format

The compiler uses a rustc-inspired error format designed for readability and tool integration. Each error includes an error code, source location, and the relevant source line with a caret pointing to the error.

5.1. Basic Error Format

error[E0008]: undefined identifier: x
 --> MyFile.z:5:10
     |
   5 |     print x
     |           ^

The first line contains the error code (E0008) and the error message. The second line shows the file path, line number, and column number in the standard File:line:col format used by gcc, clang, and rustc. The source line is displayed with a caret marker pointing to the exact location of the error.

5.2. Notes and Hints

Some errors include additional context as notes (explaining why the error occurred) or hints (suggesting how to fix it):

error[E0200]: function returns 'borrow' but has no 'lock' parameter
 --> MyFile.z:3:1
     |
   3 | f: function out string.borrow is { return "hello" }
     | ^
     = hint: add .lock to a parameter to borrow from it
error[E0008]: undefined identifier: pritn
 --> MyFile.z:2:5
     |
   2 |     pritn "hello"
     |     ^^^^^
     = hint: did you mean 'print'?

5.3. Error Code Numbering

Error codes are grouped by category:

  • E0001–E0099: Parser and syntax errors — malformed source code, unexpected tokens, missing delimiters, invalid definitions
  • E0100–E0199: Type resolution errors — type mismatches, undefined types, incompatible assignments
  • E0200–E0299: Ownership errors — invalid take/borrow/lock usage, use after ownership transfer, lock violations, reftype aliasing
  • E0300–E0399: Call and argument errors — wrong number of arguments, named argument mismatches, type mismatches in function calls
  • E0400–E0499: Generic and monomorphization errors — missing type arguments, failed type inference, invalid generic constraints

An error count summary is printed at the end:

3 errors found

6. Compiling the Output

The compiler produces a standalone C source file. To compile it into an executable, use a C compiler:

zc hello --src examples
gcc -Wall -Wno-unused-function -o hello hello.c
./hello

For development and testing, compile with address sanitizer to detect memory errors:

gcc -fsanitize=address,undefined -fno-omit-frame-pointer -Wall -Wno-unused-function -o hello hello.c
./hello

7. Examples

Compile a simple program:

zc hello --src examples

Compile with verbose output and custom output path:

zc myapp --src src -o build/myapp.c -v

Compile and run:

zc hello --src examples && gcc -o hello hello.c && ./hello

Generate SQL diagnostics:

zc hello --src examples --dump-sql - | sqlite3 :memory:

Full type-check of a library (not just reachable from main):

zc mylib --src lib --full-typecheck

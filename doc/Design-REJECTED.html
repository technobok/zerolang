6. Why no 'cast'?

A cast keyword was considered then rejected.

Was to be used for:

  1. casting to a protocol supported by an object (for the remainder of the scope only)

    Instead, the object must list labelled protocols in an as clause. The expression must then use dotted notation on the source to cast to that named protocol. The source (if a named source) is exclusively locked while the new protocol object is in scope. The protocol object is borrowed from the source object.

    Why is this better? The supported protocols are discoverable from the parent object (via autocompletion or documentation on the parent).

    To make an 'owned' protocol instance from an owned base object, you must use the constuctor of the protocol. It's compiler generated 'create' method has a 'from' argument that will 'take' a union of all of the object that impliment this method.

  2. converting an underlying type to a typedef

    Instead, the typedef has a compiler generated constructor created that will 'borrow' the underlying object for the remainder of the scope. Outside the scope, the original variable (if a named variable and if it was not given away in the innder scope) will be accessible as its original type. How to 'take'? There is no conversion, only a type change.

  3. converting an underlying type from a typedef

    Instead, this is not possible. Create child scopes to convert to a typedef, and let the inner scope end to get back to the underlying variable of the original type.

7. Why no short-circuit operations?

Operations are very simple: always left to right; alternate 'call' syntax with one argument.

So boolean 'and' and 'or' are methods on bool that always evaluate their argument.

a: 1 == 1 or (x == 1)
b: 1 == 1 and (x == 1)

But note that if will short-circuit. The first test that fails will halt execution. So, short-circuit 'and' (ie. fail on first false) is accomplished with consecutive 'when' clauses:

# safe is a bool that prevents dangerousfunc from running (when false)
a: if safe when dangerousfunc then true else false

Short-circuit 'or' (pass on first true) must be done with separate when/then clauses.

# picks first non-zero item from first, second, otherthing (default)
a: if first != 0 then first when second != 0 then second else otherthing

9. NOPE: Old section from spec about: Views

Views are a set of method/field names that can be used to control the exposed interface of objects.

Views are always a flat set of identifiers:

  • single identifiers only (no dotted identifiers) that must be refer to declarations within the object body
  • if a subview is included in a view, that subview identifier plus all members of that subview are added to the view being defined
  • an identifier may be included more than once without error (ie. perhaps via common children in included subviews), subsequent inclusions are ignored (it is a set)
  • views may be defined directly as a view argument to an object: this limits the namespace of that object to that view (ie. the children available directly via the defined object name). Usually used to control static methods, constructors etc
  • views defined directly on object namespaces will also be used for limiting available children on instances by default. Although, this can be overridden by specifying this.viewname as the return value for each of the constructors
  • typedef's may ONLY limit with a view specified directly on the typedef definition as an argument. This will then be applied to instances of that typedef (it cannot be overridden because it is forbidden to have constructors for typedefs)
  • If a this.viewname is specified for one constructor, it must be specified for all constructors.
  • views can be defined in the body of an object
  • views defined in the body can be referenced as subviews in the namespace view or in any other view defined in the body or on methods
  • views can be specified with a 'view' argument to function definitions of methods (only). This will limit the parent ('this') object to that view after the method returns. This may require runtime support to record this state.
  • views can be specified in external function definitions (as typename.viewname) provided the view name is exposed. This will limit that function body to only those methods/fields listed in the view specified

16. Why are Enumerations vs Unions different things?

  • otherwise would be jamming too many concepts together
  • if not then how to represent a union's descriminator? Another simpler union?
  • if not then how to specify descriminator type for a union? Need some clumsy syntax? Enum syntax is simpler
  • enumerations are homogenous. Could do that with union but would need to scan each subtype to ensure it is all same type. Using a union seems clumsier
  • constructing a union member in a consistent way would make the syntax overcomplicated for a simple enum. Better to separate enums (accessing simple constant values by dot notation) and unions (constructing complex types or assigning them into the union)
  • it would be confusing to have both dot notation (for the descriminator) and constructor notation for the union itself. And how would you specify the type of the descriminator?

Downside for Enumerations: accessing them in a module requires 2 dots. This is mitigated by using parameter namespaces if the enum will only be passed to functions. But still might want to store them and then supply them to a function.... Should prefer to 'import' the parent item and refer to subvalues from there in that case.

x: mymod.myenum.myvalue

myenum: mymod.myenum
x: myenum.myvalue

22. IDEA: how to make a single function that takes a generic numeric item?

Without specialising it. eg a function that can operate on an array of any size.

To fix, define the length as a variable of the type of the generic length variable. Then the compiler enforces that var must be a constant (exactly the same as the generic length 'type') and, since the generic type is a constant, this variable will have a default value equal to the length so need not be separately specified in the caller. Then refer to that variable in the body of the function and the generated function will be deduplicated.

# function is generic over array length 'type' (lt)... will be specialised 
# for all lengths...
# lt = generic length 'type' (constant)
# arr = the array itself declared in relation to the length lt
# l = an actual (runtime) parameter of type lt, so a constant

f: function {lt: int.generic arr: (array u32 to: l) l: lt} is {
    for x: arr {
        # ...
    }
        # l can be used as a constant value anywhere
        print l
        x: l + 1
}

# referencing 'lt' is the same a referencing a literal int BUT will be 
# different for each specialisation (same as being replaced with each 
# different constant). This will not be deduplicated

# BUT, defining an actual parameter as for 'l' above and then referencing
# that WILL be deduplicated because all function AST's will be the same
# (a reference to a parameter name)

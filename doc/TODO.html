myiterabletype: class {
  field: u32
  field2: u8
} as {

  create: function {field: u32 field2: u8} out this is {
    # ...
    t: meta.create :field :field2
    return t
  }

  iterate: function { t: this.lock ivar: u8 } out mytypeiterator is {
    # 'this' in 'in' parameter MUST be locked to return the iterator below
    # (because it holds a lock over 'this' so must escape into an env that
    # also locks 'this')
    # so mytypeiterator returned holds lock over 'this'

    # iterator being returned is owned, so must be a localvar/takenparam
    # 't' is locked after this line
    i: mytypeiterator t.private ivar: ivar

    return i
  }

  iterate2: function { t: this.lock } out itersubtype is {
    # can have other iterator functions, but those not named 'iterate'
    # would need to be called explicitly
    # can also have no additional args in an iterator function
  }
}

mytypeiterator: class {
  # all iterators must be classes (reftypes) No, why?
  # the iterator (likely) needs to hold a reference to the iterable, so 
  #  this one needs to be a class to hold a reference to another class 
  #  instance
  state: int  # fields to maintain state of iterator here
  target: myiterabletype.private # do we need to specify private here?
} as {

  # lock on 'target' type is required so it can be stored as a field
  create: function {target: mytype.private.lock ivar: u8 } out this is {

    # return type could be 'taken' (if a new object is constructed - BUT

For:

The labelled target of a for (if present) is examined:

- if it is a function or an object method, call it every iteration - NOTE: the function/method is NOT immediately evaluated - if it is an object instance with an iterate method, call that to get the iterator, then call that iterator on every iteration - if it is an object instance with a call method (with or without arguments), call that every iteration (with the supplied arguments) - ownership: the for exposes the correct ownership since the compiler will check the types in the 'call' method of the iterator. If a taken type is returned, it will be owned in the body of the for. If borrowed, it will be borrowed in the body. - scoping in the body ends at the end of the loop or immediately after a break or continue. So a lock on the iterator held by the returned element will be released immediately before the start of the next loop so that the iterator is able to be called to return the next element (which itself will then take a lock for the loop body)

for x: myfunc loop {
  # x is set to myfunc result before every iteration
}

for x: myfunc2 myarg loop {
  # x is set to myfunc result before every iteration
  # function can have arguments
}

for x: (myfunc3 myarg name: mynamedarg) loop {
  # x is set to myfunc result before every iteration
  # function can have arguments, named arguments must be in parens
}

myobject: make_an_object_with_a_call_method
for x: myobject loop {
  # x is set to myobject.call result before every iteration
}

myobject2: make_an_object_with_a_call_method_with_args
for x: myobject2 myarg loop {
  # x is set to myobject2.call result before every iteration
  # 'call' can have arguments, 
  # named arguments must be in parens (not shown)
}

10. this type meta

type - (meta.type ?) when inside a type definition, this is a synonym for the type name. Useful for self referencing in methods and fields. Instances of this type have access to the *public* API of that instance only (no private method/field access).

this - when inside a type definition, this is *this* instance currently running the method. 'this' is of type 'type' but it is more specific. There is only this 'this' :). This instance of 'this' has access to the *full* API (public and private methods/fields)

meta - when inside a type definition, 'meta' provides access to meta fields and methods related to this type definition. The one true constructor created by the compiler for this object is here as meta.create. meta.name has the fully qualified name of this type (?). Other methods and fields?

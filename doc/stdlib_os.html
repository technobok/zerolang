1. Overview

The os unit exposes process-level primitives: argv, environment variables, working directory, process identity, and platform / architecture descriptors. Paired with io -- where io surfaces failures as Result t: T e: IoError, os primitives are either infallible or use Option when absence is expected.

Accessed via qualified names only -- os.args, os.env ..., os.exit ...; there is no unqualified re-export.

Design principles.

  • Option vs. Result by semantic. If absence is normal, return Option(T) (env). If a syscall can fail, return Result t: T e: IoError (setEnv, cwd). No separate os.error -- errno domain is shared with io, and callers often interleave env mutation with file I/O
  • Ambient process state is implicit. os.args, os.cwd, os.pid take no handle. Mirrors io.stdin / io.stdout / io.stderr
  • Caller owns returned heap. Every function returning a String or List yields freshly-allocated storage. Scope-exit runs the destructor
  • POSIX first. Natives use libc: setenv, unsetenv, environ, getcwd, chdir, getpid, getppid, getpwuid_r, gethostname. Windows stubs are a v2 concern

3. args

  args: function out (List of: String) is native

Program argument vector as passed to main. Position 0 is the program name (as the kernel / shell provided it); subsequent elements are the explicit arguments.

The returned list and every string inside are freshly-allocated and owned by the caller; scope exit frees both. Repeated calls each re-allocate -- cache the result if you iterate multiple times.

  a: os.args
  for arg: a.listview loop { print arg }

4. get_env

  env: function {key: StringView} out (Option t: String) is native

Look up environment variable key. Returns Option.some value when the variable is set (even to the empty string); Option.none when the variable is not present.

Missing-env is Option (not IoError) because it is an expected everyday condition.

  match (os.env key: "HOME")
  case some then use some
  case none then use "/tmp"

5. set_env

  setEnv: function {key: StringView value: StringView} out (Result t: null e: IoError) is native

Insert or overwrite an environment variable in the current process. setenv(key, value, 1) under the hood -- always overwrites.

Errors surface as IoError; the most common is other (EINVAL on a malformed key). The errno domain is shared with io since env mutation often interleaves with file I/O.

6. unset_env

  unsetEnv: function {key: StringView} out (Result t: null e: IoError) is native

Remove key from the environment. Succeeds whether or not key was previously set; errors only on an invalid key name (EINVAL surfaces as IoError.other).

7. env_names

  envNames: function out (List of: String) is native

Snapshot of every environment variable name in the current process, in environ order. The returned list and each of its entries are freshly-allocated; scope exit frees both.

Values are omitted from the snapshot to avoid an O(n) string copy for callers that filter first; use os.env to fetch each value.

  names: os.envNames
  for n: names.listview loop {
      match (os.env key: n)
      case some then { print n; print some }
      case none then {}    # raced; skip
  }

8. cwd

  cwd: function out (Result t: String e: IoError) is native

Current working directory as an absolute path. Typically succeeds; errors only when the cwd inode has been unlinked or permission to resolve it has been revoked.

9. set_cwd

  setCwd: function {path: StringView} out (Result t: null e: IoError) is native

chdir to Path. Affects subsequent relative-path I/O calls. Fails with the usual IoError variants (notfound, notdir, permissiondenied).

10. pid

  pid: function out i32 is native

Current process id. POSIX pid_t is int on Linux; narrowed to i32. Infallible.

11. ppid

  ppid: function out i32 is native

Parent process id. Infallible.

12. user_name

  userName: function out (Result t: String e: IoError) is native

Login name of the effective user, resolved via the passwd database (getpwuid_r(geteuid())). Errors when the uid has no passwd entry or the database is unreadable.

13. home_dir

  homeDir: function out (Result t: String e: IoError) is native

The effective user's home directory. Reads $HOME from the environment; errors with IoError.other when unset. Does not fall back to the passwd database -- set $HOME explicitly for portable behaviour across stripped environments.

14. hostname

  hostname: function out (Result t: String e: IoError) is native

Result of gethostname(2), truncated at HOST_NAME_MAX. Result-wrapped because the syscall can fail in rare cases.

15. platformkind

  platformkind: variant {
      linux:   null
      darwin:  null
      windows: null
      other:   null
  }

Host OS family. Exposed as a runtime variant for forward-compat with cross-compilation; the initial backend resolves it at compile time via the C preprocessor.

16. archkind

  archkind: variant {
      x86_64:  null
      aarch64: null
      other:   null
  }

Host CPU architecture family.

17. platform

  platform: function out platformkind is native

Value of the current host's platformkind.

  match (os.platform)
  case linux   then ...
  case darwin  then ...
  case windows then ...
  case other   then ...

18. arch

  arch: function out archkind is native

Value of the current host's archkind.

19. exit

  exit: function {code: i32} is native

Terminate the process immediately with code as the status. Does not return; all open file descriptors are flushed by libc's atexit chain, but zerolang-level destructors for in-scope values do not run.

Reserve for fatal-error exits and CLI tools that want an explicit status. Prefer a normal return path for everyday success / failure so RAII cleanup fires.

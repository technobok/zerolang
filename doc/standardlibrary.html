4. array

A valtype. A fixed-length, mutable, homogenous block of valtype items. Zero-initialized on creation. Length is part of the type and cannot be changed.

Generic over element type (of) and length (to). Different element types or lengths produce different monomorphized types (e.g. array_i64_4, array_i64_8). Defined as a record in collections.z and aliased in core.z as array: collections.array.

~of specifies the element type. Must be a valtype. ~to specifies the array length as a compile-time constant.

Construction: (array of: <type> to: <length>) creates a zero-initialized array. Elements can also be initialized from a data block via data_name.array.

Compile-time indexed access uses dotted numeric syntax: a.0, a.1, etc. Out-of-bounds compile-time indices produce a type error.

Methods (runtime, zero-indexed):

  • get i: index — returns element at index; runtime error if out of bounds
  • set i: index val: new — replaces element, returns old value; runtime error if out of bounds
  • length — u64, compile-time constant equal to array size

Since arrays are valtypes, they are copied on assignment and when passed to functions.

5. list

Create/specify a list. A reftype. A growable, heap-allocated, homogenous collection of items.

Mutable. Can be extended or reduced (has capacity and length for efficient size adjustments). Length is NOT part of the type; it is a runtime parameter.

Generic over type of elements only. Defined as a class in collections.z and aliased in core.z as list: collections.list.

~of specifies the element type. Different element types produce different monomorphized list types (e.g. list_i64, list_u64).

Construction: (list of: <type>) creates an empty list. An optional capacity: argument pre-allocates storage: (list of: i64) capacity: 10.u64.

In memory, a list is a pointer to a struct with capacity, length, and a data pointer. Growth formula: newcap = oldcap + (oldcap >> 1) + 4.

Methods (all zero-indexed):

  • append from: elem — takes ownership, adds to end, grows if needed
  • insert from: elem at: index — takes ownership, shifts elements right, grows if needed
  • extend from: other_list — takes ownership of source list, bulk copies elements, frees source container
  • get i: index — returns element at index; runtime error if out of bounds
  • set i: index val: new — replaces element, returns old value; runtime error if out of bounds
  • pop — returns last element; runtime error if empty
  • length — u64, current number of elements
  • capacity — u64, current allocated capacity

When the list falls out of scope it is destroyed: each reftype element is freed, then the data buffer and struct are freed.

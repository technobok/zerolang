1. Overview

The zerolang standard library ships as six units under lib/system/. Each unit is documented on its own page; this index lists the units and summarises what ships in each.

Every symbol in every unit is listed in that unit's Index section near the top of the page, with a one-line signature and a link to the full detail section. Scan the index to find a name; click through for prose, ownership notes, and examples where they clarify use.

2. Units

  • system -- numeric types (i8.. i128, u8.. u128, f32, f64), bool, null, never, Any; string types (String, StringView, str); option / result / optionval / parseerror; bytes / byteview / path / pathview; box / typedef / tag; control-flow natives
  • collections -- array, List, ListView, Map generics plus the stringJoin helper
  • io -- stream protocols (Reader, Writer, Closer, Seeker); File / openmode / seekorigin; filesystem ops; buffered readers / writers; print / eprintln / the standard streams
  • os -- argv, environment, working directory, process identity, platform / arch, exit
  • cli -- argv parsing: Spec, Parsed, flag / option / positional definitions, parse, helpText

3. The core unit

lib/system/core.z is an alias layer, not an API surface. It re-exports a small curated subset of the other units under unqualified names so everyday code doesn't need an explicit system. / io. / collections. prefix. The re-exports include:

  • Numeric types -- i8.. i128, u8.. u128, f32, f64
  • Primitives -- bool, null, never, Any, String, StringView
  • Generic containers -- Option, optionval, Result, Box, array, str, List, ListView, Map
  • Typedefs -- Bytes, ByteView, Path, PathView, typedef
  • Protocols -- Reader, Writer, Closer, Seeker, Text, StringLike
  • File types -- File, openmode, seekorigin, IoError
  • Standard streams -- stdin, stdout, stderr
  • Free functions -- print, stringJoin
  • Control flow -- error, panic, return, break, continue

The io, os, and cli units are not unqualified -- code must write io.open ..., os.args, cli.parse ... (matching the filesystem / process convention used in other languages that expose these as namespaces rather than globals).
